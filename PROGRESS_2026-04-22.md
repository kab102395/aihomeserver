## 3) How the system works (current architecture)

### Core state machine
The server runs a task through these nodes:

1. **Intake**
   - Creates a task id + loads conversation context
   - Sets workspace, risk gate config, etc.

2. **Planner**
   - Produces a JSON plan: a list of steps
   - Each step can be either:
     - tool step (`tool_binding` set), or
     - pure LLM step (`tool_binding: null`)

3. **Executor**
   - Advances `current_step`
   - If itŌĆÖs a tool step: generates a **tool call JSON** (the ŌĆ£what to runŌĆØ payload)
   - If itŌĆÖs an LLM-only step: generates text (or JSON when `output_format: "json"`)
   - Enforces `requires_facts` for grounded research

4. **ToolExecution**
   - Dispatches the tool call into the tool registry
   - Saves tool outputs (including failures) into artifacts under `*_result`
   - Emits SSE events for the UI (tool start, terminal output, file written, etc.)

5. **Critic**
   - Evaluates whether the last step succeeded
   - Deterministically fails if the tool step failed, so Repair/Replan can kick in

6. **Repair**
   - If the last step was a tool step: regenerates a corrected tool call JSON and retries the tool
   - If it was an LLM step: regenerates corrected text
   - Repair is bounded; repeated failure triggers a replan

7. **Replan**
   - Generates a new plan when repair is exhausted

8. **Finalization**
   - Summarizes result state + stores conversation history

### Artifacts (the ŌĆ£working memoryŌĆØ of a run)
Artifacts are a map of keys ŌåÆ JSON values used to:

- pass tool outputs into later steps
- store the extracted `facts` table for grounded code generation
- capture failures for debugging (status codes, error traces, etc.)

### Tools (what the agent can actually do)
Key tools now:

- `filesystem`: read/write/list + new `find`/`grep`
- `shell`: run a command in the runtime OS
- `git`: status/log/commit helpers
- `parallel_search`, `web_search`: web search
- `http_fetch`: fetch + sanitize pages for evidence
- `save_knowledge`: persist learned material

### Web UI
The UI shows:

- A Files tab with:
  - directory tree
  - editor
  - upload/download
